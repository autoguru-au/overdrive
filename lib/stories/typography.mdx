# Typography

Overdrive typography follows the AutoGuru Design System 2026. The typeface is
**Averta Std** (`AvertaStandard, system-ui, sans-serif`) with letter-spacing `0`
everywhere.

The 2026 styles ship as **new, opt-in named styles** — the legacy numeric size
scale (`'1'`–`'9'`) is untouched, so existing screens don't change. Adopt the
named styles screen by screen.

Line heights of the named styles are ratio-derived:

- **Headings** use a `1.25` ratio — tight, for scannable hierarchy.
- **Paragraphs** use a `1.4` ratio — comfortable for reading and dense UI.

## Named styles

Named styles are the preferred way to pick a size in new and updated designs.
They are accepted by the `size` prop of `<Text>`, `<Heading>`, and the
`textStyles` / `typography` functions.

| Named style | px / line-height | Weights                                 |
| ----------- | ---------------- | --------------------------------------- |
| `h1`        | 40 / 50          | bold (700), regular (400)               |
| `h2`        | 32 / 40          | bold (700), regular (400)               |
| `h3`        | 24 / 30          | bold (700), regular (400)               |
| `h4`        | 20 / 25          | bold (700), regular (400)               |
| `p1`        | 16 / 22.4        | regular (400), semiBold, bold           |
| `p2`        | 14 / 19.6        | regular (400), semiBold, bold           |
| `p3`        | 12 / 16.8        | regular (400), semiBold, bold           |
| `p4`        | 10 / 14          | regular (400), bold (700) — no semiBold |

Named styles apply their default weight (bold for headings, regular for
paragraphs) unless `weight` or `strong` is set.

```tsx
<Heading as="h2" size="h2">2026 H2 — 32px, bold</Heading>
<Heading as="h2" size="h2" weight="normal">The H2 regular variant</Heading>
<Text size="p2">Dense UI body copy</Text>
<Text size="p3" color="soft">A caption</Text>
```

Without an explicit `size`, `<Heading>` keeps its legacy sizes (h1 30px, h2
24px, h3 20px, h4 16px) — pass the matching named style to opt a heading into
the 2026 hierarchy.

## Usage rules

**Headings** — use `h1`–`h4` for the page hierarchy. `h1` is reserved for the
page title (one per page); `h2` for major sections; `h3`/`h4` for sub-sections
and card titles. Named heading styles default to bold; use the regular (400)
variant via `weight="normal"` for large display text that shouldn't shout. The
2026 hierarchy stops at `h4` — prefer restructuring content over inventing
deeper levels.

**Body** — `p1` (16px) is the default reading size for content pages and
long-form copy. In dense SaaS screens (tables, forms, dashboards, admin
tooling), use `p2` (14px) as the working body size — it keeps rows compact
while staying readable at a 1.4 line-height.

**Labels** — form and field labels use `p2` with `semiBold` weight.

**Captions & helper text** — use `p3` (12px) for hint text, timestamps,
metadata, and table footnotes, typically in a soft colour.

**Fine print** — `p4` (10px) is for legal lines, badges, and micro-labels only.
Never use `semiBold` at this size (it is not part of the spec) and avoid
paragraphs of it.

**Weights** — regular (400) for prose, `semiBold` for emphasis, labels, and
interactive text, bold (700) for headings and strong emphasis. Don't use weight
alone to create hierarchy across more than two levels — change size too.
